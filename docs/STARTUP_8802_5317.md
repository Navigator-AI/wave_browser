# Wave Browser Startup Guide (Backend 8802, Frontend 5317)

This guide explains how to start the project locally with:
- Backend: `8802`
- Frontend: `5317`

## Prerequisites

- Python 3.10+ (or available `python3`)
- Node.js 18+
- npm

## 1) Start Backend on 8802

Open a terminal at the project root and run:

```bash
./scripts/start-backend.sh --host 0.0.0.0 --port 8802
```

Notes:
- The script creates `backend/venv` automatically if missing.
- Backend API base URL: `http://localhost:8802`
- Swagger docs: `http://localhost:8802/docs`

Optional (auto-reload for development):

```bash
./scripts/start-backend.sh --host 0.0.0.0 --port 8802 --reload
```

## 2) Start Frontend on 5317

Open a second terminal at the project root and run:

```bash
cd frontend
npm install
npm run dev -- --host 0.0.0.0 --port 5317
```

Frontend URL:

```text
http://localhost:5317
```

## 3) Open Frontend and Point to Backend

Use this URL so the frontend talks to backend `8802`:

```text
http://localhost:5317?server=localhost:8802
```

## 4) Accessing the Backend

### API Documentation

Once the backend is running, view interactive API docs:

```text
http://localhost:8802/docs
```

This Swagger UI shows all available endpoints with request/response schemas.

### Main Backend Endpoints

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/health` | GET | Health check |
| `/api/sessions` | GET | List all sessions |
| `/api/sessions` | POST | Create new session |
| `/api/sessions/{session_id}` | GET | Get session details |
| `/api/upload` | POST | Upload Verilog files (multi-file) |
| `/api/simulate` | POST | Run simulation on uploaded files |
| `/api/files/open/{path}` | POST | Open waveform file (VCD/FSDB) |

### Example: Upload Verilog Files

```bash
curl -X POST http://localhost:8802/api/upload \
  -F "file=@example/rtl/counter.v" \
  -F "file=@example/rtl/tb_counter.v"
```

Response:
```json
{
  "files": [
    "/projects/sujayak/wave_browser/uploads/counter.v",
    "/projects/sujayak/wave_browser/uploads/tb_counter.v"
  ]
}
```

### Example: Run Simulation

```bash
curl -X POST http://localhost:8802/api/simulate \
  -H "Content-Type: application/json" \
  -d '{
    "files": [
      "/projects/sujayak/wave_browser/uploads/counter.v",
      "/projects/sujayak/wave_browser/uploads/tb_counter.v"
    ]
  }'
```

Response:
```json
{
  "vcd_path": "/tmp/wave_sim_xyz123/output.vcd",
  "files": [...]
}
```

## 5) Accessing the Frontend

### Main URL (with Backend Connection)

```text
http://localhost:5317?server=localhost:8802
```

This connects the frontend to the backend at port 8802.

### URL Parameters

| Parameter | Purpose | Example |
|-----------|---------|---------|
| `server` | Backend host:port | `?server=localhost:8802` |
| `fsdb` | Auto-load FSDB file on startup | `&fsdb=/path/to/file.fsdb` |

### Example: Full URL with Parameters

```text
http://localhost:5317?server=localhost:8802&fsdb=/path/to/dump.fsdb
```

## 6) Complete Workflow: Upload & Simulate

### Step 1: Open File Dialog

- Click "Open" button in frontend (top toolbar)
- Select one or more Verilog `.v` files
- Files are uploaded to backend `/api/upload`

### Step 2: Create Session

- After upload, frontend automatically creates a session
- Files are stored in `uploads/` directory

### Step 3: Run Simulation (for Verilog files)

- If Verilog files detected, "Run Simulation" button appears
- Click button to compile and simulate
- Backend processes:
  - Converts FSDB dump calls to VCD (if needed)
  - Auto-generates testbench (if missing)
  - Compiles with `iverilog -g2012`
  - Runs simulation with `vvp`
  - Returns `output.vcd` waveform

### Step 4: View Waveform

- Waveform automatically loads in timeline
- Pan, zoom, and select signals to inspect
- Displays VCD signal values correctly

## 7) Feature Reference

### Waveform Features

- **Timeline Navigation**: Scroll to pan, zoom to change scale
- **Signal Inspector**: Click signal name to see values
- **Multi-file Support**: Upload RTL + testbench together
- **Auto-testbench**: System generates testbench if only design provided
- **FSDB→VCD**: Automatic conversion during simulation

### File Types Supported

| Type | Source | Usage |
|------|--------|-------|
| `.vcd` | Direct upload or simulation output | Waveform display |
| `.fsdb` | Direct upload | Waveform display (if Verdi available) |
| `.v` / `.sv` | Direct upload | Verilog RTL simulation |

## 8) Project Structure

| `backend/` | FastAPI server (port 8802) |
| `backend/app/routers/` | API endpoint definitions |

## 9) Quick Troubleshooting

### Backend Not Responding

- Confirm backend running: `http://localhost:8802/docs`
- Check terminal for error messages
- Ensure port 8802 is not in use: `lsof -i :8802` (macOS/Linux)

### Frontend Cannot Connect to Backend

- Use correct URL: `http://localhost:5317?server=localhost:8802`
- Confirm `?server=` parameter is present
- Check browser console (F12) for errors

### Port Already in Use

Stop conflicting process:
```bash
# Find process on port
lsof -i :8802
lsof -i :5317

# Kill process (replace PID with actual process ID)
kill -9 <PID>
```

### Rebuild Backend Environment

```bash
cd backend
rm -rf venv
python3 -m venv venv
./venv/bin/pip install -r requirements.txt
```

### Rebuild Frontend Environment

```bash
cd frontend
rm -rf node_modules package-lock.json
npm install
```

## 10) Development Notes

### Auto-reload Backend Changes

For development, use `--reload` flag:
```bash
./scripts/start-backend.sh --host 0.0.0.0 --port 8802 --reload
```

### Mock Frontend Mode

To test frontend without backend running:
```bash
cd frontend
npm run dev -- --host 0.0.0.0 --port 5317
# Open: http://localhost:5317 (without ?server param)
```

### Running Tests

**Backend:**
```bash
cd backend
pytest
```

**Frontend:**
```bash
cd frontend
npm test
```
